# Stacktrace — Agent Briefing (DISC.md)

Read this first. It's the context another agent (or human) needs to work on
this project without re-explaining everything.

## What it is

**Stacktrace** is a native macOS app (SwiftUI) for **daily work journaling** —
*"show progress, not hours."* It is deliberately **not** a time tracker. The
point is to capture *what you did, how it went, what went well, what didn't*,
and your general wellbeing — then turn that into clean reports for managers.

Local-first and private: all data lives in a JSON file on the user's Mac.
Nothing leaves the machine except features the user explicitly opts into
(optional OpenAI text cleanup; optional team sync).

Public repo: https://github.com/wpoortman/stacktrace (MIT). Default branch
`main`.

## Core concepts

- **Entry kinds** (all are `ReportEntry`, distinguished by fields):
  - Full entry — title, description, what went well, what went bad/improve, tags, a 1–5 **mood**.
  - Quick **win** / **setback** — one-liner (`quickKind`).
  - **Check-in** — mood only (one tap).
  - **Exercise** — name + minutes (movement, not billable time).
  - **Meeting reflection** — from the calendar (`eventID`), happened? + went well/bad.
- **Day score** — an overall 1–10 rating of the whole day (separate from per-entry mood).
- **Routines** — recurring movement reminders (daily or hourly with interval + weekdays), checked off on the Activity tab.
- **Holiday mode** — periods where the app stops all nudges.

## Main UI

Sidebar tabs (macOS Settings-style icons): **Dashboard** (greeting, day-rating
prompt, streak/stats cards, GitHub-style contribution graph coloured by
mood/score), **Activity** (exercise + routines), **Meetings** (review calendar
meetings), **Trends** (Swift Charts; Pro-gated), **Exports** (saved PDFs).
Plus a **menu-bar** quick-capture popover.

Reports: pick a date range → **PDF** (grouped by day, iconed items) or **Copy
as Markdown** (for Slack/standup). Each day also has a one-click export. PDFs
live in an app-managed Exports folder; share via the share sheet.

Settings is a grouped sidebar: **App** (General, Storage, Pro, Team), **Logging**
(Days, Tags), **Schedule** (Routines, Calendar, Reminders, Holiday), **Assist** (AI).

## Architecture

- **Storage:** `DataStore` (ObservableObject, `@MainActor`, `@EnvironmentObject`)
  persists to `~/Library/Application Support/Stacktrace/data.json` (atomic
  write + `.bak`; resilient/optional decoding so upgrades never wipe data).
  Location is user-configurable (`StorageLocation`, security-scoped bookmark) —
  pointing it at an iCloud Drive folder gives cross-Mac sync.
  NOTE: when the app is **sandboxed** (any signed build) that path resolves to
  the app's container, not the literal `~/Library/Application Support`.
- **PDF:** `ReportHTMLBuilder` → HTML, `PDFReportGenerator` renders via
  `WKWebView.pdf(...)` sliced into A4 pages with PDFKit. Do **not** use
  `NSPrintOperation` — it runs away/hangs on this macOS.
- **Reminders:** `NotificationManager` (UNUserNotificationCenter, local).
- **AI integration:** `mcp/` is a Node MCP server exposing tools (add entry /
  win / setback / exercise / set day score / today) that read-modify-write the
  same `data.json`. `FileWatcher` makes `DataStore` live-reload external writes.
  Sandbox caveat: a signed app's data is in its container — point the server at
  the app's storage folder (env `STACKTRACE_DIR`/`STACKTRACE_DATA`).
- **Calendar:** `CalendarService` (EventKit) — reads macOS Calendar, which
  already aggregates Google/iCloud/Exchange, so no per-provider OAuth.
- **Licensing (Pro):** `Licensing.swift` — `ProManager` gates Pro features via a
  signed, offline-verified entitlement (CryptoKit). `LicenseService` protocol
  with a local `MockLicenseService` (dev keys: any text = Individual, `TEAM-…`
  = 50 seats, `CUSTOM-…`). Real keys come from a license backend (Lemon
  Squeezy/Paddle) later.
- **Team:** `TeamAPI.swift` — client for the future agency backend
  (`docs/API.md`). `MockTeamAPI` offline; `HTTPTeamAPI` skeleton; opt-in.

## Tech / repo conventions

- SwiftUI, macOS 14+ target, no third-party dependencies.
- Xcode project is **generated by XcodeGen from `project.yml`** — after adding/
  moving files or changing settings, run `xcodegen generate` and commit.
- Source in `Stacktrace/`, tests in `StacktraceTests/` (XCTest, ~31 tests).
- Type named `ReportEntry` (not `Entry` — collides with SwiftUI's `@Entry`).
  `ReportView`/`ReportHTMLBuilder` mean the work-*report* PDF, not the app.

## Build / test / run

```bash
xcodegen generate
xcodebuild -project Stacktrace.xcodeproj -scheme Stacktrace -configuration Debug \
  build CODE_SIGNING_ALLOWED=NO
xcodebuild test -project Stacktrace.xcodeproj -scheme Stacktrace \
  -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO ENABLE_DEBUG_DYLIB=NO
```

CI (`.github/workflows/ci.yml`) runs the tests on every push/PR.

**Signing gotcha:** notifications, login-item, and calendar work reliably only
in a **signed** build. The dev build is unsigned; to test those, ad-hoc sign:
`codesign --force --deep --sign - --entitlements Stacktrace/Stacktrace.entitlements <App>`.
Ad-hoc signing turns the **sandbox on**, which moves the data path to the
container — `DataStore` migrates/relocates, but be aware of it.

## Where it's going

- **Pro tier** (yearly license, seat-based): unlock advanced features (Trends,
  etc.). One app — gated by entitlement, distributed direct (Developer ID
  notarized) for seat licensing; App Store build would use individual IAP.
- **Agency / Team backend (planned, Laravel — "part c", not built yet):** a web
  admin where an agency owner manages members, **roles with global hourly
  rates** (Developer 80, Designer 90…), and per-member overrides. The app pushes
  coarse daily summaries (wins/losses, wellbeing, day score) via the API in
  `docs/API.md`; the backend computes an **effective billable rate =
  baseRate × qualityFactor(wellbeing/day-scores)**. Idea: a rough patch hurts
  the agency's margin, not the client. **Ethics rule:** this wellbeing→pay link
  must stay opt-in, transparent to the employee, and coarse/aggregated.
- App Store readiness done: privacy manifest, encryption flag, MIT license.

## For the agent building the Laravel API/admin (part c)

The macOS app already speaks this contract — it currently runs against an
offline `MockTeamAPI`; setting a **Team server URL** in Settings → Team flips it
to the real `HTTPTeamAPI`. Your job is the backend it points at.

**Source of truth for the wire format: [`docs/API.md`](docs/API.md).** Build to
that exactly (field names, casing, JSON shapes). Summary:

- `GET /v1/me` → `{ memberId, name, role, currency, baseRateCents,
  effectiveRateCents, qualityFactor }`.
- `POST /v1/metrics` (idempotent per member+date) body `{ date(yyyy-MM-dd),
  entries, wins, losses, wellbeing(1–5|null), dayScore(1–10|null) }` →
  `{ stored, effectiveRateCents }`.
- optional `GET /v1/metrics?from&to`.
- All JSON. Auth header `Authorization: Bearer <seatToken>`. `401` = bad/revoked
  seat → app prompts re-activation. Errors: `{ "error": "..." }`.

**Auth / licensing:** the user buys a license (Individual = 1 seat, Team = 50,
Custom = >50) — use a provider like **Lemon Squeezy/Paddle** (handles keys,
seat caps, tax) or roll your own. On activation (`activate(key, deviceID)`) the
backend must consume a seat and return a **seat token** the app stores and
sends as the Bearer. The token maps to `(agency, member, device)`.

**Suggested data model (multi-tenant):**
- `agencies` — the tenant (the agency owner's org).
- `roles` — `name`, `base_rate_cents` (per-agency global rate per function).
- `members` — `agency_id`, `name`, `role_id`, optional `rate_override_cents`.
- `licenses` / `seats` / `activations` — key, plan, seat count, device bindings.
- `daily_metrics` — `member_id`, `date`, `entries`, `wins`, `losses`,
  `wellbeing`, `day_score`, `effective_rate_cents` (unique on member+date).

**Rate engine (server-side, keep it here so it's tunable without an app
release):**
```
baseRateCents      = member.rate_override ?? role.base_rate
qualityFactor      = f(recent wellbeing + day scores)   // 0..1, your formula
effectiveRateCents = round(baseRateCents * qualityFactor)
```
The app ships a reference `qualityFactor` (0.7–1.0) in `TeamMetrics` for the
mock — yours can differ; the server value is authoritative and returned in
`/v1/me` and `/v1/metrics`.

**Admin web UI (Laravel, browser):** agency owner logs in; CRUD members, roles
+ global rates, per-member overrides; dashboards; client-facing rate/billing
reports. The macOS app never shows other members — it only reads its own
`/v1/me` and pushes its own metrics.

**Non-negotiable privacy/ethics:** only the coarse daily numbers above ever
leave a member's Mac (never note text), sync is opt-in, and the wellbeing→rate
adjustment must be transparent to the employee. Honor that server-side too
(don't store more than the contract sends; log/justify rate changes).
